fitnesse-maven-runner-plugin

fitnesserunner:run

Full name:

com.github.andreptb:fitnesse-maven-runner-plugin:0.2.1:run

Description:

Plugin allow configuration to run FitNesse, resembling regular FitNesse command line but providing extra features such as classpath configuration and so on.

Attributes:

  • Requires a Maven project to be executed.
  • Requires dependency resolution of artifacts in scope: test.

Optional Parameters

Name Type Since Description
command String - If informed, will run the command and then exit, useful to run tests and then exit. For example: mvn fitnesserunner:run -Dfitnesse.command=SuiteToRun?suite&format=text
Check here for all available options. Note that only resource?responder&inputs part of the URL needs be informed
User property is: fitnesse.command.
contextRoot String - Allows setting a custom contextRoot to the application: http://[host][contextRoot]
User property is: fitnesse.contextRoot.
fitNesseRoot String - The directory in which FitNesse looks for top level pages.
Default value is: FitNesseRoot.
User property is: fitnesse.fitNesseRoot.
fitnesseClasspathVariable String - Creates a variable to be used by fitnesse containing the classpath available along the run command.
Default value is: FITNESSE_CLASSPATH.
User property is: fitnesse.classpath.variable.
includeProjectDependencies boolean - Indicates if the project dependencies should be used when executing the main class.
Default value is: false.
User property is: fitnesse.includeProjectDependencies.
port Integer - FitNesse port to start in WEB mode. Defaults to 8000
User property is: fitnesse.port.
redirectOutput String - Redirect command output. This is most useful in conjunction with the command option
User property is: fitnesse.redirectOutput.
rootPath String - The directory in which FitNesse expects to find its page root.
User property is: fitnesse.rootPath.
theme String - Allows setting a custom theme
User property is: fitnesse.theme.
